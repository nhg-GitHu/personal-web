# 个人网站项目 - 实现思想与逻辑流程

## 一、项目概述

这是一个基于 Next.js 的个人技术网站，支持 6 个内容板块（AI / 系统 / 软件 / NAS / Docker / 项目），项目板块带评论功能。网站采用 MDX 作为内容格式，部署到 Vercel，视觉风格为科技美感 + 浅色/深色模式切换。

## 二、技术栈

| 层级 | 技术 | 作用 |
|------|------|------|
| 框架 | Next.js 16 (App Router) | 全栈框架，支持 SSG/SSR |
| 样式 | Tailwind CSS v4 | 原子化 CSS 框架 |
| 内容 | MDX (Markdown) | 文章格式，通过 remark 渲染为 HTML |
| 评论 | Giscus | 基于 GitHub Discussions 的评论系统 |
| 搜索 | 客户端搜索 | 构建时生成索引，浏览器端匹配 |
| 部署 | Vercel | 零配置部署，自动 HTTPS |
| 图标 | 内联 SVG | 北极熊 Logo 和社交图标 |

## 四、核心逻辑流程

### 4.1 内容管理系统

**流程：写文章 → 读取 → 渲染**

```
用户在 src/content/[category]/ 下新建 .mdx 文件
    ↓
posts.ts 扫描目录，读取文件
    ↓
gray-matter 解析 frontmatter（标题、日期、摘要等元数据）
    ↓
unified → remark-parse → remark-gfm → remark-rehype → rehype-stringify
    ↓
Markdown 被转换为 HTML
    ↓
文章页通过 dangerouslySetInnerHTML 渲染 HTML
```

**关键文件：**
- `src/lib/posts.ts` — 提供 `getPostContent()`、`getAllPosts()`、`getPostsByCategory()` 等函数
- `src/content/[category]/xxx.mdx` — 文章源文件

**Frontmatter 格式：**
```yaml
---
title: "中文标题"
titleEn: "English Title"
description: "文章摘要"
date: "2026-05-06"
---
```

### 4.2 路由系统

**Next.js App Router 动态路由：**

```
/                          → 首页（展示所有板块 + 最新文章）
/[category]                → 板块列表页（如 /ai、/system）
/[category]/[slug]         → 文章详情页（如 /ai/what-is-llm）
/sitemap.xml               → 站点地图（SEO 收录）
```

**`generateStaticParams()` 的作用：**
构建时扫描所有文章，生成静态路由列表，确保每个板块和文章都有对应的静态 HTML。

### 4.3 评论系统

```
用户访问 /projects/xxx 文章页
    ↓
GiscusComments.tsx 检测 pathname 是否以 /projects/ 开头
    ↓
如果是，渲染 Giscus 组件
    ↓
Giscus 通过 GitHub API 查找对应的 Discussion
    ↓
将 Discussion 内容作为评论展示
    ↓
用户用 GitHub 账号登录后可以评论
    ↓
评论写入 GitHub Discussions
```

**Giscus 配置（GiscusComments.tsx）：**
- `repo`：nhg-GitHu/personal-web
- `repoId`：R_kgDOSVb0Vw
- `category`：General
- `categoryId`：DIC_kwDOSVb0V84C8arY
- `mapping`：pathname（每篇文章对应一个 Discussion）

### 4.4 搜索系统

```
页面加载时，Header 接收所有文章的 PostMeta 数组
    ↓
用户点击搜索按钮或按 Ctrl+K
    ↓
SearchDialog 打开，显示搜索输入框
    ↓
用户输入关键词
    ↓
客户端过滤：遍历所有文章，匹配 title 和 description
    ↓
显示匹配结果（文章标题 + 板块名）
    ↓
点击结果跳转到对应文章页
```

**特点：** 纯客户端实现，不需要额外的搜索服务，构建时生成索引 JSON。

### 4.5 深色模式

```
页面加载时，检测 document.documentElement 的 dark class
    ↓
Header 中的 toggleDark 函数切换 dark class
    ↓
Tailwind CSS 的 .dark 选择器匹配
    ↓
:root 和 .dark 中定义的 CSS 变量自动切换
    ↓
页面颜色和样式跟随变化
```

**样式定义（globals.css）：**
```css
:root {
  --background: #ffffff;
  --foreground: #171717;
}
.dark {
  --background: #0a0a0a;
  --foreground: #ededed;
}
```

## 五、数据流

```
用户访问 → Next.js SSR/SSG 生成 HTML → 返回页面
    ↓
客户端交互（搜索、深色模式切换）→ React Client Components
    ↓
添加文章 → git add → git commit → git push
    ↓
GitHub Webhook 触发 Vercel 重新构建
    ↓
Vercel 拉取最新代码 → npm run build → 部署新版本
```

## 六、部署流程

```
1. 代码推送到 GitHub (nhg-GitHu/personal-web)
    ↓
2. Vercel 检测到新推送（自动触发）
    ↓
3. Vercel 执行 npm run build
    ↓
4. 读取 src/content/ 下所有文章，生成静态 HTML
    ↓
5. 部署到 Vercel CDN
    ↓
6. 用户访问 xxx.vercel.app 看到最新内容
```

## 九、添加新文章

1. 在对应板块的 `src/content/[category]/` 目录下新建 `.mdx` 文件
2. 文件名为文章 URL slug（如 `what-is-llm.mdx`）
3. 文件顶部写 frontmatter（标题、英文标题、描述、日期）
4. 下方写 Markdown 正文（支持 GFM 语法、代码块、图片）
5. 图片使用外部图床 URL（如 NAS Web 服务器）
6. 执行 `git add -A && git commit -m "add: xxx" && git push origin main`
7. Vercel 自动重新构建并部署

## 十、技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 文章存储 | MDX 文件 | 简单，零依赖，写文章就是写 Markdown |
| 渲染方式 | remark → HTML | 避免 @mdx-js/react 与 React 19 兼容性问题 |
| 评论系统 | Giscus | 基于 GitHub Discussions，免费开源 |
| 搜索 | 客户端搜索 | 文章量不大，不需要额外搜索服务 |
| 部署 | Vercel | Next.js 原生支持，零配置部署 |
| 图片 | 外部图床 | 不增加项目体积，NAS 可做图床 |
