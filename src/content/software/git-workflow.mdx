# Git 工作流最佳实践

良好的 Git 工作流是团队协作的基础。

## 推荐分支策略

```
main (production)
├── develop (integration)
│   ├── feature/user-auth
│   ├── feature/payment
│   └── bugfix/login-error
├── release/v1.0
└── hotfix/security-patch
```

## 提交信息规范

使用 [Conventional Commits](https://www.conventionalcommits.org/) 格式：

```bash
feat: add user authentication
fix: resolve login error on mobile
docs: update API documentation
```
